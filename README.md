## SQL HANDS -ON- PRACTICE
# Employee Database Queries
## Overview
This guided exercise walks you through writing SQL queries on an employee database. Starting from basic filtering and progressing to complex, multi-condition logic, you'll answer real-world business questions step by step.

# Levels
Level 1: Basic Filtering
Simple WHERE queries to get comfortable.

Task	Description
* High Earners:	Find employees earning ≥ 120,000
* Specific Identification:	Retrieve full details for employee_number = 1005
*	List everyone in the IT department
Level 2: Smart Logic
Combine conditions using AND, OR, BETWEEN, and NOT.

Task	Description
* Female employees in Finance
* Salaries between 70,000 and 90,000
* Employees who are not Active (e.g., On Leave, Resigned)
Level 3: Pattern Matching
Use LIKE, IN, and wildcard searches.

Task	Description
* Job title contains "Manager" anywhere
* Employees in Sales, Marketing, or Operations (IN operator)
* Lookup	First name starts with "A"
 Combined Query
Apply all concepts in a single query.

Find all Male employees who:

* Work in Sales OR IT

* Were hired after 2015-01-01

* Earn more than 80,000
